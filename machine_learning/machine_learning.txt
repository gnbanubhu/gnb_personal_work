--------------------------------------------------------------------------------
1. WHAT IS MACHINE LEARNING?
--------------------------------------------------------------------------------
Machine Learning (ML) is a subset of Artificial Intelligence (AI) where systems
learn from data to improve their performance on tasks without being explicitly
programmed. Instead of writing rules manually, the model discovers patterns from
examples.

    Traditional Programming:  Data + Rules  --> Output
    Machine Learning:         Data + Output --> Rules (Model)


--------------------------------------------------------------------------------
2. TYPES OF MACHINE LEARNING
--------------------------------------------------------------------------------

2.1 SUPERVISED LEARNING
-----------------------
The model learns from labeled data (input-output pairs).

  Goal: Learn a mapping function f(X) = Y

  Types:
    - Classification  : Predict a discrete label
                        Example: Email spam detection (spam / not spam)
    - Regression      : Predict a continuous value
                        Example: House price prediction

  Common Algorithms:
    - Linear Regression
    - Logistic Regression
    - Decision Trees
    - Random Forest
    - Support Vector Machines (SVM)
    - K-Nearest Neighbors (KNN)
    - Neural Networks
    - Gradient Boosting (XGBoost, LightGBM)


2.2 UNSUPERVISED LEARNING
--------------------------
The model learns from unlabeled data by finding hidden patterns or structure.

  Types:
    - Clustering          : Group similar data points
                            Example: Customer segmentation
    - Dimensionality Reduction : Reduce number of features
                            Example: PCA, t-SNE
    - Association         : Find rules between variables
                            Example: Market basket analysis

  Common Algorithms:
    - K-Means Clustering
    - DBSCAN
    - Hierarchical Clustering
    - Principal Component Analysis (PCA)
    - Autoencoders


2.3 SEMI-SUPERVISED LEARNING
-----------------------------
Uses a small amount of labeled data combined with a large amount of unlabeled
data during training.

  Example: Image classification where only a few images are labeled.


2.4 REINFORCEMENT LEARNING
---------------------------
An agent learns by interacting with an environment, receiving rewards or
penalties based on actions taken.

  Key Concepts:
    - Agent      : The learner/decision maker
    - Environment: What the agent interacts with
    - State      : Current situation of the agent
    - Action     : Choices available to the agent
    - Reward     : Feedback signal (positive/negative)
    - Policy     : Strategy the agent follows

  Example: Game playing (AlphaGo), robotics, self-driving cars


--------------------------------------------------------------------------------
3. KEY ML CONCEPTS
--------------------------------------------------------------------------------

3.1 FEATURES AND LABELS
------------------------
  - Feature (X)  : Input variable / independent variable
  - Label (Y)    : Output variable / dependent variable / target
  - Feature Engineering: Creating new features from raw data


3.2 TRAINING, VALIDATION, AND TEST SETS
-----------------------------------------
  - Training Set   : Used to train the model (typically 60-80%)
  - Validation Set : Used to tune hyperparameters (typically 10-20%)
  - Test Set       : Used to evaluate final model performance (typically 10-20%)


3.3 OVERFITTING AND UNDERFITTING
----------------------------------
  - Overfitting  : Model learns training data too well, including noise.
                   High training accuracy, low test accuracy.
                   Fix: More data, regularization, simpler model, dropout

  - Underfitting : Model is too simple to capture patterns.
                   Low training accuracy and low test accuracy.
                   Fix: More complex model, more features, less regularization

  - Bias-Variance Tradeoff:
      High Bias   --> Underfitting (model too simple)
      High Variance --> Overfitting (model too complex)
      Goal: Find the sweet spot with low bias and low variance


3.4 REGULARIZATION
-------------------
Techniques to prevent overfitting by penalizing model complexity.

  - L1 Regularization (Lasso) : Adds |weights| penalty. Can zero out features.
  - L2 Regularization (Ridge) : Adds weights^2 penalty. Shrinks weights.
  - Elastic Net               : Combination of L1 and L2
  - Dropout                   : Randomly disables neurons during training (NNs)


3.5 CROSS-VALIDATION
---------------------
Technique to evaluate model generalization on unseen data.

  - K-Fold Cross-Validation: Split data into K folds. Train on K-1 folds,
    validate on the remaining fold. Repeat K times, average the results.


3.6 HYPERPARAMETERS vs PARAMETERS
-----------------------------------
  - Parameters     : Learned by the model during training (e.g., weights, biases)
  - Hyperparameters: Set before training by the developer
                     (e.g., learning rate, number of trees, max depth)

  Hyperparameter Tuning:
    - Grid Search   : Try all combinations
    - Random Search : Try random combinations
    - Bayesian Optimization: Smart search based on past results


--------------------------------------------------------------------------------
4. MODEL EVALUATION METRICS
--------------------------------------------------------------------------------

4.1 CLASSIFICATION METRICS
----------------------------
  - Accuracy    = (TP + TN) / Total
  - Precision   = TP / (TP + FP)   -- Of predicted positives, how many are correct?
  - Recall      = TP / (TP + FN)   -- Of actual positives, how many did we find?
  - F1 Score    = 2 * (Precision * Recall) / (Precision + Recall)
  - AUC-ROC     : Area Under the ROC Curve (1.0 = perfect, 0.5 = random)
  - Confusion Matrix: Table showing TP, TN, FP, FN

  TP = True Positive   FP = False Positive (Type I Error)
  TN = True Negative   FN = False Negative (Type II Error)


4.2 REGRESSION METRICS
------------------------
  - MAE  (Mean Absolute Error)      : Average of |actual - predicted|
  - MSE  (Mean Squared Error)       : Average of (actual - predicted)^2
  - RMSE (Root Mean Squared Error)  : sqrt(MSE)
  - R²   (R-Squared / Coefficient of Determination): 1 = perfect fit, 0 = baseline


--------------------------------------------------------------------------------
5. COMMON ALGORITHMS IN DETAIL
--------------------------------------------------------------------------------

5.1 LINEAR REGRESSION
----------------------
  Models the linear relationship between input features and a continuous output.

  Equation: Y = w0 + w1*X1 + w2*X2 + ... + wn*Xn

  Loss Function: Mean Squared Error (MSE)
  Optimization: Gradient Descent or Normal Equation


5.2 LOGISTIC REGRESSION
------------------------
  Used for binary classification. Applies sigmoid function to output probability.

  Sigmoid: sigma(z) = 1 / (1 + e^(-z))
  Output: Probability between 0 and 1
  Loss Function: Binary Cross-Entropy


5.3 DECISION TREES
-------------------
  A tree-like model where each internal node represents a feature, each branch
  represents a decision rule, and each leaf represents an outcome.

  Splitting Criteria:
    - Gini Impurity (Classification)
    - Information Gain / Entropy (Classification)
    - Variance Reduction (Regression)

  Pros: Interpretable, handles non-linear data
  Cons: Prone to overfitting


5.4 RANDOM FOREST
------------------
  Ensemble of many decision trees trained on random subsets of data (Bagging).
  Final prediction is the majority vote (classification) or average (regression).

  Key Concepts:
    - Bootstrap Sampling : Random sampling with replacement
    - Feature Randomness : Each tree uses a random subset of features
    - Reduces overfitting compared to a single decision tree


5.5 GRADIENT BOOSTING
----------------------
  Builds trees sequentially where each new tree corrects errors of the previous.

  Popular Implementations:
    - XGBoost   : Extreme Gradient Boosting (fast, regularized)
    - LightGBM  : Light Gradient Boosting Machine (very fast, memory efficient)
    - CatBoost  : Handles categorical features natively


5.6 SUPPORT VECTOR MACHINES (SVM)
-----------------------------------
  Finds the optimal hyperplane that best separates classes with maximum margin.

  Key Concepts:
    - Support Vectors : Data points closest to the decision boundary
    - Margin          : Distance between decision boundary and support vectors
    - Kernel Trick    : Maps data to higher dimensions (RBF, Polynomial kernels)

  Works well for high-dimensional data and small datasets.


5.7 K-NEAREST NEIGHBORS (KNN)
-------------------------------
  Classifies a data point based on the majority class of its K nearest neighbors.

  - No training phase (lazy learner)
  - Distance metrics: Euclidean, Manhattan, Minkowski
  - Sensitive to irrelevant features and scale (requires normalization)


5.8 K-MEANS CLUSTERING
-----------------------
  Partitions data into K clusters by minimizing within-cluster variance.

  Steps:
    1. Initialize K centroids randomly
    2. Assign each point to the nearest centroid
    3. Recompute centroids as the mean of assigned points
    4. Repeat steps 2-3 until convergence

  Choosing K: Elbow Method, Silhouette Score


--------------------------------------------------------------------------------
6. DATA PREPROCESSING
--------------------------------------------------------------------------------

6.1 HANDLING MISSING VALUES
-----------------------------
  - Remove rows/columns with missing data
  - Imputation: Replace with mean, median, mode, or a predicted value


6.2 FEATURE SCALING
--------------------
  Algorithms sensitive to scale (KNN, SVM, Neural Networks) require scaling.

  - Standardization (Z-score): X' = (X - mean) / std       --> Range: (-inf, +inf)
  - Min-Max Scaling           : X' = (X - min) / (max - min) --> Range: [0, 1]
  - Robust Scaling            : Uses median and IQR, robust to outliers


6.3 ENCODING CATEGORICAL VARIABLES
------------------------------------
  - Label Encoding    : Assigns integer to each category (0, 1, 2 ...)
  - One-Hot Encoding  : Creates binary column per category
  - Ordinal Encoding  : For ordered categories (low, medium, high)


6.4 FEATURE SELECTION
----------------------
  - Filter Methods    : Statistical tests (correlation, chi-square)
  - Wrapper Methods   : Recursive Feature Elimination (RFE)
  - Embedded Methods  : LASSO, feature importance from tree models


6.5 HANDLING IMBALANCED DATA
------------------------------
  - Oversampling  : SMOTE (Synthetic Minority Over-sampling Technique)
  - Undersampling : Reduce majority class samples
  - Class Weights : Penalize misclassification of minority class


--------------------------------------------------------------------------------
7. DIMENSIONALITY REDUCTION
--------------------------------------------------------------------------------

7.1 PCA (PRINCIPAL COMPONENT ANALYSIS)
----------------------------------------
  Transforms features into a set of linearly uncorrelated components (principal
  components) ordered by the amount of variance they explain.

  Use: Reduce features, remove multicollinearity, speed up training.


7.2 t-SNE (t-Distributed Stochastic Neighbor Embedding)
---------------------------------------------------------
  Non-linear dimensionality reduction for visualization of high-dimensional data.
  Preserves local structure (similar points stay close).

  Use: Visualizing clusters in 2D or 3D. Not suitable for feature reduction.


7.3 LDA (LINEAR DISCRIMINANT ANALYSIS)
----------------------------------------
  Supervised dimensionality reduction. Maximizes class separability.
  Projects data onto axes that maximize separation between classes.


--------------------------------------------------------------------------------
8. NEURAL NETWORKS (BASICS)
--------------------------------------------------------------------------------

  A neural network is a series of layers of interconnected nodes (neurons).

  Structure:
    - Input Layer   : Receives raw features
    - Hidden Layers : Learn intermediate representations
    - Output Layer  : Produces final prediction

  Key Concepts:
    - Activation Functions : Introduce non-linearity
        ReLU    : max(0, x)          -- Most common in hidden layers
        Sigmoid : 1 / (1 + e^-x)    -- Binary classification output
        Softmax : Multi-class output probabilities
        Tanh    : Range (-1, 1)

    - Backpropagation : Algorithm to compute gradients and update weights
    - Gradient Descent: Optimization algorithm to minimize loss
        - Batch Gradient Descent
        - Stochastic Gradient Descent (SGD)
        - Mini-batch Gradient Descent

    - Learning Rate : Controls step size in weight updates
                      Too high = diverge, Too low = slow convergence

    - Epochs        : One full pass through the entire training dataset
    - Batch Size    : Number of samples per gradient update

9. ML WORKFLOW / PIPELINE
--------------------------------------------------------------------------------

  Step 1 : Define the Problem
            - What are we predicting? Classification or Regression?
            - What is the success metric?

  Step 2 : Collect and Understand Data
            - Gather data from various sources
            - Exploratory Data Analysis (EDA)

  Step 3 : Data Preprocessing
            - Handle missing values, outliers
            - Feature engineering and scaling

  Step 4 : Split Data
            - Train / Validation / Test split

  Step 5 : Choose and Train Model
            - Start simple (Linear/Logistic Regression)
            - Iterate to more complex models

  Step 6 : Evaluate Model
            - Use appropriate metrics
            - Cross-validation

  Step 7 : Hyperparameter Tuning
            - Grid Search / Random Search / Bayesian Optimization

  Step 8 : Deploy Model
            - Serve predictions via API (Flask, FastAPI)
            - Monitor model performance in production

  Step 9 : Monitor and Retrain
            - Watch for data drift and model degradation
            - Retrain periodically with new data

11. IMPORTANT TERMINOLOGY QUICK REFERENCE
--------------------------------------------------------------------------------

  Term                  | Description
  ----------------------|------------------------------------------------------
  Epoch                 | One full pass over training data
  Batch                 | Subset of training data used per iteration
  Learning Rate         | Step size for weight updates
  Loss Function         | Measures model error (MSE, Cross-Entropy)
  Gradient Descent      | Optimization to minimize loss
  Backpropagation       | Compute gradients through the network
  Regularization        | Technique to prevent overfitting
  Dropout               | Randomly deactivate neurons during training
  Embedding             | Dense vector representation of data
  Transfer Learning     | Reuse pre-trained model on new task
  Ensemble              | Combine multiple models for better performance
  Bagging               | Train models on random data subsets (Random Forest)
  Boosting              | Train models sequentially correcting errors (XGBoost)
  Stacking              | Use predictions of models as input to another model
  Data Augmentation     | Artificially increase training data size
  Feature Importance    | Measure of how useful a feature is in the model
  Confusion Matrix      | Table of actual vs predicted class counts
  ROC Curve             | Plot of True Positive Rate vs False Positive Rate
  Precision-Recall Curve| Useful for imbalanced classification problems
