HOW TO CREATE NLP MODELS?
==========================

OVERVIEW
---------
Creating an NLP model means building a system that can understand,
process, or generate human language. The approach depends on the task,
available data, compute budget, and required accuracy.

There are four main paths to creating NLP models:

  Path 1 — Use a Pre-trained Model Off-the-Shelf   (easiest, fastest)
  Path 2 — Fine-tune a Pre-trained Model           (most practical)
  Path 3 — Train a Model from Scratch              (rare, expensive)
  Path 4 — Use an LLM via API (zero/few-shot)      (no training needed)

This guide covers all four paths with step-by-step instructions.


END-TO-END NLP MODEL CREATION PIPELINE
----------------------------------------

  Step 1  →  Define the Problem & Task
  Step 2  →  Collect & Prepare Data
  Step 3  →  Text Preprocessing
  Step 4  →  Feature Engineering / Tokenization
  Step 5  →  Choose Model Architecture
  Step 6  →  Train the Model
  Step 7  →  Evaluate the Model
  Step 8  →  Tune & Improve
  Step 9  →  Deploy the Model
  Step 10 →  Monitor & Maintain


STEP 1 — DEFINE THE PROBLEM & TASK
------------------------------------
Before writing any code, clearly define:

  Questions to answer:
    • What NLP task is this? (classification, NER, generation, etc.)
    • What is the input? (single sentence, document, conversation)
    • What is the expected output? (label, entity span, generated text)
    • How much labeled data do you have?
    • What are the latency and cost constraints?
    • Does it need to run offline or can it use a cloud API?

  NLP Task Types:
    Text Classification      → assign labels to text
    Token Classification     → label each token (NER, POS)
    Text Generation          → produce new text
    Seq2Seq                  → transform input text to output text
    Question Answering       → find/generate an answer
    Semantic Similarity      → score similarity between two texts
    Summarization            → condense long text
    Translation              → convert between languages

  Example problem definition:
    Task     : Classify customer support tickets by department
    Input    : Free-form ticket text (1–500 words)
    Output   : One of [Billing, Technical, General, Returns]
    Data     : 5,000 labeled historical tickets
    Latency  : < 200ms per request
    Deploy   : REST API on cloud


STEP 2 — COLLECT & PREPARE DATA
---------------------------------
Data is the most critical ingredient for NLP models.

  Data Sources:
    Internal data     — existing labeled records, databases, logs
    Web scraping      — BeautifulSoup, Scrapy for public text data
    Public datasets   — Hugging Face datasets, Kaggle, UCI, arXiv
    Annotation tools  — Label Studio, Prodigy, Doccano, Scale AI
    Data augmentation — paraphrase, back-translation, synonym replacement
    LLM generation    — generate synthetic labeled examples with GPT/Claude

  Data Requirements by Task:
    Text Classification     500–50,000+ examples per class
    NER                     1,000–10,000+ annotated sentences
    Text Generation (SFT)   1,000–100,000+ instruction-response pairs
    Summarization           500–10,000+ document-summary pairs
    Translation             100,000–millions of sentence pairs

  Data Quality Checklist:
    ✔  Label consistency — same text always gets same label
    ✔  Class balance — avoid extreme imbalance (use oversampling/undersampling)
    ✔  Representative — covers all real-world input variations
    ✔  Clean — remove duplicates, corrupted records, mislabeled examples
    ✔  Split — train / validation / test (e.g., 80% / 10% / 10%)

  Code example — load data with Hugging Face datasets:
    from datasets import load_dataset
    dataset = load_dataset("imdb")          # public dataset
    dataset = load_dataset("csv", data_files={"train": "train.csv",
                                               "test":  "test.csv"})


STEP 3 — TEXT PREPROCESSING
------------------------------
Clean and normalize raw text before feeding it to a model.

  Preprocessing steps:
    1. Lowercase           "Hello World" → "hello world"
    2. Remove HTML/URLs    "<p>text</p>" → "text"
    3. Remove special chars "price: $10!" → "price 10"
    4. Expand contractions "can't" → "cannot"
    5. Remove stopwords    "the cat sat" → "cat sat"   (optional)
    6. Stemming            "running" → "run"           (optional)
    7. Lemmatization       "better" → "good"           (optional)
    8. Handle encoding     Fix UTF-8, remove null bytes

  When to apply each step:
    Lowercase        — almost always (unless case matters: NER, proper nouns)
    HTML/URL removal — web text, email, social media
    Stopword removal — TF-IDF / classical ML (NOT for transformers)
    Stemming/Lemma   — classical ML (NOT needed for transformers)
    Special chars    — depends on task (keep punctuation for sentiment)

  Code example:
    import re, nltk
    from nltk.corpus import stopwords
    from nltk.stem import WordNetLemmatizer

    stop_words = set(stopwords.words("english"))
    lemmatizer = WordNetLemmatizer()

    def preprocess(text):
        text = text.lower()
        text = re.sub(r"<.*?>", " ", text)          # remove HTML
        text = re.sub(r"http\S+", " ", text)        # remove URLs
        text = re.sub(r"[^a-z\s]", " ", text)       # keep letters only
        tokens = text.split()
        tokens = [lemmatizer.lemmatize(t) for t in tokens
                  if t not in stop_words]
        return " ".join(tokens)


STEP 4 — FEATURE ENGINEERING / TOKENIZATION
---------------------------------------------
Convert text into numerical representations the model can process.

  Approach A — Classical ML Features (for non-transformer models)
    TF-IDF Vectorization
      from sklearn.feature_extraction.text import TfidfVectorizer
      vectorizer = TfidfVectorizer(max_features=10000, ngram_range=(1, 2))
      X_train = vectorizer.fit_transform(train_texts)
      X_test  = vectorizer.transform(test_texts)

    Bag of Words (CountVectorizer)
      from sklearn.feature_extraction.text import CountVectorizer
      vectorizer = CountVectorizer(max_features=5000)
      X = vectorizer.fit_transform(texts)

    Word Embeddings (average Word2Vec / GloVe vectors per document)
      import numpy as np
      def doc_vector(text, model):
          words = [w for w in text.split() if w in model]
          return np.mean([model[w] for w in words], axis=0) if words else np.zeros(300)

  Approach B — Transformer Tokenization (for BERT, RoBERTa, etc.)
    from transformers import AutoTokenizer
    tokenizer = AutoTokenizer.from_pretrained("bert-base-uncased")
    encoded = tokenizer(
        texts,
        padding=True,
        truncation=True,
        max_length=512,
        return_tensors="pt"
    )
    # Returns: input_ids, attention_mask, token_type_ids

  Approach C — Subword Tokenization (for custom LLM training)
    from tokenizers import ByteLevelBPETokenizer
    tokenizer = ByteLevelBPETokenizer()
    tokenizer.train(files=["corpus.txt"], vocab_size=32000, min_frequency=2)
    tokenizer.save_model("my_tokenizer/")


STEP 5 — CHOOSE MODEL ARCHITECTURE
-------------------------------------
Select the right model based on task, data size, and constraints.

  Option A — Classical ML Models (fast, interpretable, small data OK)
    Naive Bayes           — text classification baseline, fast
    Logistic Regression   — strong baseline for classification
    SVM (LinearSVC)       — excellent for text classification
    Random Forest         — ensemble, handles non-linear patterns
    XGBoost               — strong for structured NLP features

    Best for: classification with TF-IDF features, fast prototyping

  Option B — Pre-trained Transformer Models (best general choice)
    BERT / DistilBERT     — bidirectional, classification, NER, QA
    RoBERTa               — better trained BERT, strong classification
    DeBERTa               — SOTA for many classification benchmarks
    ALBERT                — lightweight BERT variant
    XLNet                 — autoregressive BERT alternative

    Best for: classification, NER, semantic similarity, extractive QA

  Option C — Seq2Seq / Encoder-Decoder Models
    T5 / mT5              — text-to-text, summarization, translation, QA
    BART                  — denoising pre-training, excellent summarization
    MarianMT              — Helsinki-NLP, machine translation
    Pegasus               — abstractive summarization

    Best for: summarization, translation, question generation

  Option D — Decoder-Only / Generative Models
    GPT-2                 — text generation, fine-tunable
    LLaMA / Mistral       — open-source LLMs, instruction following
    Claude / GPT-4        — API-based, best quality, zero/few-shot

    Best for: text generation, dialogue, instruction following

  Option E — Specialized Models
    sentence-transformers — semantic similarity, embeddings
    CrossEncoder          — re-ranking, pairwise similarity scoring
    Whisper               — speech-to-text
    spaCy pipelines       — fast production NLP (NER, POS, parsing)

  Model Selection Guide:
    Task                   Recommended Model
    ─────────────────────────────────────────────────────────────
    Text classification    DistilBERT, RoBERTa, DeBERTa
    Token classification   BERT, RoBERTa (with token classification head)
    Semantic similarity    sentence-transformers (all-MiniLM-L6-v2)
    Summarization          BART, T5, PEGASUS
    Translation            MarianMT, Helsinki-NLP models
    Question answering     RoBERTa-base-squad2, DeBERTa-v3
    Text generation        GPT-2, LLaMA, Mistral (or LLM API)
    NER                    spaCy, BERT-NER, custom fine-tuned


STEP 6 — TRAIN THE MODEL
--------------------------

  Path A — Classical ML Training
    from sklearn.linear_model import LogisticRegression
    from sklearn.pipeline import Pipeline
    from sklearn.feature_extraction.text import TfidfVectorizer

    pipeline = Pipeline([
        ("tfidf", TfidfVectorizer(max_features=10000, ngram_range=(1,2))),
        ("clf",   LogisticRegression(max_iter=1000, C=1.0))
    ])
    pipeline.fit(X_train, y_train)
    predictions = pipeline.predict(X_test)

  Path B — Fine-tuning with Hugging Face Trainer
    from transformers import (AutoTokenizer, AutoModelForSequenceClassification,
                              TrainingArguments, Trainer)
    from datasets import Dataset

    # Load model
    model_name = "distilbert-base-uncased"
    tokenizer  = AutoTokenizer.from_pretrained(model_name)
    model      = AutoModelForSequenceClassification.from_pretrained(
                     model_name, num_labels=4)

    # Tokenize data
    def tokenize(batch):
        return tokenizer(batch["text"], padding=True,
                         truncation=True, max_length=512)
    dataset = Dataset.from_dict({"text": texts, "label": labels})
    dataset = dataset.map(tokenize, batched=True)

    # Training arguments
    args = TrainingArguments(
        output_dir        = "./results",
        num_train_epochs  = 3,
        per_device_train_batch_size = 16,
        per_device_eval_batch_size  = 32,
        learning_rate     = 2e-5,
        weight_decay      = 0.01,
        evaluation_strategy = "epoch",
        save_strategy     = "epoch",
        load_best_model_at_end = True,
        metric_for_best_model  = "f1"
    )

    # Train
    trainer = Trainer(model=model, args=args,
                      train_dataset=train_ds, eval_dataset=val_ds,
                      compute_metrics=compute_metrics)
    trainer.train()
    trainer.save_model("./my_model")

  Path C — LoRA / QLoRA Fine-tuning (parameter-efficient)
    from peft import LoraConfig, get_peft_model, TaskType
    from transformers import AutoModelForCausalLM

    model = AutoModelForCausalLM.from_pretrained(
        "meta-llama/Llama-3-8b", load_in_4bit=True)

    lora_config = LoraConfig(
        r=16, lora_alpha=32,
        target_modules=["q_proj", "v_proj"],
        lora_dropout=0.05,
        task_type=TaskType.CAUSAL_LM
    )
    model = get_peft_model(model, lora_config)
    model.print_trainable_parameters()
    # Trainable: ~0.1% of total parameters

  Key Training Hyperparameters:
    Learning rate        2e-5 to 5e-5 (fine-tuning), 1e-4 (training from scratch)
    Batch size           16–64 (GPU memory dependent)
    Epochs               2–5 for fine-tuning
    Warmup steps         10% of total steps
    Weight decay         0.01
    Optimizer            AdamW
    Scheduler            Linear decay with warmup, cosine decay
    Max sequence length  128–512 for classification, up to 4096 for generation


STEP 7 — EVALUATE THE MODEL
-----------------------------
Choose metrics that align with your task and business objective.

  Classification Metrics:
    from sklearn.metrics import classification_report, confusion_matrix
    print(classification_report(y_true, y_pred))
    # Shows: precision, recall, F1-score per class + macro/weighted avg

  NER Metrics:
    from seqeval.metrics import classification_report
    report = classification_report(true_labels, pred_labels)
    # Entity-level F1, not token-level

  Semantic Similarity:
    from scipy.stats import pearsonr, spearmanr
    pearson_corr, _  = pearsonr(human_scores, model_scores)
    spearman_corr, _ = spearmanr(human_scores, model_scores)

  Summarization Metrics:
    from rouge_score import rouge_scorer
    scorer = rouge_scorer.RougeScorer(["rouge1", "rouge2", "rougeL"])
    scores = scorer.score(reference_summary, generated_summary)

  Generation Metrics:
    BLEU    — n-gram overlap (translation quality)
    ROUGE   — recall-oriented (summarization quality)
    BERTScore — semantic similarity using BERT embeddings
    Perplexity — language model fluency

  Baseline Comparison:
    Always compare against:
      • Random baseline (predict majority class)
      • Rule-based baseline (keyword matching)
      • Previous model version
      • Human performance (gold standard)


STEP 8 — TUNE & IMPROVE
-------------------------
If model performance is not satisfactory, improve systematically.

  Data improvements (highest impact):
    • Add more labeled examples for underperforming classes
    • Fix mislabeled examples in training data
    • Use data augmentation: paraphrase, back-translation
    • Generate synthetic examples with LLMs
    • Clean noisy or ambiguous examples

  Model improvements:
    • Try a larger pre-trained model (BERT → RoBERTa → DeBERTa)
    • Increase training epochs (watch for overfitting)
    • Adjust learning rate (use learning rate finder)
    • Use class weights for imbalanced datasets
    • Ensemble multiple models (voting, stacking)

  Prompt improvements (for LLM-based approaches):
    • Add more few-shot examples
    • Refine system prompt with clearer instructions
    • Use chain-of-thought for complex reasoning tasks

  Regularization (if overfitting):
    • Increase dropout rate (0.1 → 0.3)
    • Add L2 weight decay
    • Reduce model complexity
    • Early stopping on validation loss

  Error analysis:
    • Manually inspect 50–100 misclassified examples
    • Find patterns: long texts, specific domains, ambiguous labels
    • Add targeted training examples for failure modes


STEP 9 — DEPLOY THE MODEL
---------------------------
Package and serve the trained model for production use.

  Option A — REST API with FastAPI
    from fastapi import FastAPI
    from transformers import pipeline

    app = FastAPI()
    classifier = pipeline("text-classification",
                           model="./my_model", device=0)

    @app.post("/classify")
    def classify(text: str):
        result = classifier(text)[0]
        return {"label": result["label"], "score": result["score"]}

  Option B — Batch Processing
    # For offline bulk processing
    from transformers import pipeline
    import pandas as pd

    classifier = pipeline("text-classification", model="./my_model",
                           batch_size=64, device=0)
    df = pd.read_csv("tickets.csv")
    results = classifier(df["text"].tolist())
    df["label"] = [r["label"] for r in results]
    df.to_csv("classified_tickets.csv", index=False)

  Option C — Hugging Face Hub
    trainer.push_to_hub("my-org/my-model")
    # Model is now available at huggingface.co/my-org/my-model

  Option D — ONNX / TorchScript Export (for faster inference)
    from transformers import convert_graph_to_onnx
    convert_graph_to_onnx.convert(framework="pt", model="./my_model",
                                   output=Path("model.onnx"), opset=12)

  Optimization for production:
    Quantization   — reduce model to INT8 (2–4× faster, minimal accuracy loss)
    Distillation   — compress large model into smaller student model
    Batching       — process multiple requests together for throughput
    Caching        — cache embeddings for repeated inputs
    GPU serving    — use NVIDIA Triton Inference Server for scale


STEP 10 — MONITOR & MAINTAIN
------------------------------
NLP models degrade over time as language and data distribution shift.

  Metrics to monitor:
    Accuracy / F1     — track on held-out test set periodically
    Confidence scores — flag low-confidence predictions for review
    Latency           — p50, p95, p99 response times
    Throughput        — requests per second
    Error rate        — API failures, timeouts
    Data drift        — distribution of inputs shifting over time

  Monitoring tools:
    Evidently AI    — data drift and model performance monitoring
    WhyLabs         — ML observability platform
    Arize AI        — LLM and ML monitoring
    Custom logging  — log predictions + confidence → aggregate dashboards

  Maintenance tasks:
    • Retrain periodically on new labeled data
    • Add new classes as business needs evolve
    • Review human-flagged incorrect predictions
    • Update pre-trained base model when newer versions release

SUMMARY
--------
Creating an NLP model involves 10 steps:

  1.  Define the task      — input, output, constraints, data availability
  2.  Collect data         — labeled examples, public datasets, augmentation
  3.  Preprocess text      — clean, normalize, handle noise
  4.  Featurize            — TF-IDF for classical ML, tokenizer for transformers
  5.  Choose architecture  — classical ML → BERT → LLM based on complexity
  6.  Train the model      — sklearn, Hugging Face Trainer, LoRA fine-tuning
  7.  Evaluate             — F1, ROUGE, BLEU, BERTScore, task-appropriate metric
  8.  Tune & improve       — more data, better model, hyperparameter search
  9.  Deploy               — FastAPI, Hugging Face Hub, ONNX export
  10. Monitor              — drift detection, accuracy tracking, retraining

Four practical paths:
  Path 1 — Off-the-shelf pre-trained model     (spaCy, sentence-transformers)
  Path 2 — Fine-tune a pre-trained model       (BERT, RoBERTa, LLaMA + LoRA)
  Path 3 — Train from scratch                  (rare, requires large data + GPU)
  Path 4 — LLM API with prompting              (Claude, GPT-4, zero/few-shot)

Key principle:
  Start with the simplest approach that meets your accuracy needs.
  Pre-trained + fine-tuned transformer models cover 90% of real-world
  NLP tasks without the cost and complexity of training from scratch.
