PINECONE — VECTOR DATABASE
==========================

Pinecone is a fully managed, cloud-native vector database designed for
storing, indexing, and querying high-dimensional vector embeddings at scale.
Unlike FAISS (a library you self-host), Pinecone is a SaaS product —
you don't manage infrastructure at all.

------------------------------------------------------------
WHAT IS PINECONE?
------------------------------------------------------------

Pinecone provides a managed vector search service that handles:
  - Storage of millions/billions of embeddings
  - Fast approximate nearest neighbor (ANN) search
  - Real-time updates (insert, update, delete)
  - Metadata filtering alongside vector search
  - Automatic scaling and replication

You interact with it via a simple REST API or Python/Node.js SDK.

------------------------------------------------------------
CORE CONCEPTS
------------------------------------------------------------

1. INDEX
   ------
   A Pinecone index is where vectors are stored.
   Each index has:
     - A fixed dimension (must match your embedding model)
     - A distance metric: cosine, euclidean, or dotproduct
     - A cloud region (AWS, GCP, Azure)

   Example:
     Dimension: 1536 (OpenAI text-embedding-ada-002)
     Metric: cosine

2. NAMESPACE
   ----------
   Namespaces partition data within a single index.
   Useful for multi-tenant apps (e.g., one namespace per user/org).
   Queries are scoped to a namespace.

3. VECTOR
   -------
   Each vector record has:
     - id       : unique string identifier
     - values   : the embedding (list of floats)
     - metadata : optional key-value pairs for filtering

   Example:
     {
       "id": "doc-001",
       "values": [0.12, 0.87, 0.45, ...],
       "metadata": {"source": "blog", "category": "AI", "year": 2024}
     }

4. UPSERT
   -------
   Pinecone uses "upsert" — insert if new, update if ID exists.
   This simplifies data management.

5. QUERY
   ------
   Search for top-k most similar vectors to a query vector.
   Can combine vector similarity with metadata filters.

HOW PINECONE WORKS INTERNALLY
------------------------------------------------------------

                 Your App
                    |
             Pinecone SDK/API
                    |
          ┌─────────────────────┐
          │    Pinecone Index   │
          │                     │
          │  [Vector Store]     │
          │  [ANN Index]        │  ← Approximate Nearest Neighbor
          │  [Metadata Store]   │  ← Filter by metadata
          └─────────────────────┘
                    |
            Top-K Results
          (vector IDs + scores)

Steps:
  1. Embed your data using an embedding model (OpenAI, Claude, etc.)
  2. Upsert vectors + metadata into Pinecone
  3. At query time, embed the user query
  4. Send query vector to Pinecone
  5. Pinecone returns top-K most similar vector IDs + scores
  6. Use IDs to fetch full content from your database

------------------------------------------------------------
DISTANCE METRICS
------------------------------------------------------------

Metric        | Best For                          | Formula
--------------|-----------------------------------|--------------------
cosine        | Semantic similarity (NLP/text)    | 1 - cos(A, B)
euclidean     | Geometric distance in space       | sqrt(sum((A-B)^2))
dotproduct    | Max inner product search          | A · B

Most LLM embedding use cases → use cosine similarity.

------------------------------------------------------------
METADATA FILTERING
------------------------------------------------------------

A powerful Pinecone feature: filter results by metadata BEFORE
or DURING vector search, reducing the candidate set.

Example — search only within "AI" category from 2024:

  index.query(
      vector=query_embedding,
      top_k=5,
      filter={"category": {"$eq": "AI"}, "year": {"$gte": 2024}}
  )

Supported operators: $eq, $ne, $gt, $gte, $lt, $lte, $in, $nin
