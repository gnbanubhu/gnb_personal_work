DIAGRAM 2: CUSTOMER PURCHASE JOURNEY (HAPPY PATH)
====================================================================

  CUSTOMER
     │
     │  Opens web / mobile app
     ▼
  ┌─────────────────────────────────────┐
  │         HOME PAGE / LANDING         │
  │  Personalized recommendations       │
  │  Trending products, promotions      │
  └───────────────┬─────────────────────┘
                  │  Browses / Searches
                  ▼
  ┌─────────────────────────────────────┐
  │         PRODUCT SEARCH              │
  │  Elasticsearch / Algolia            │
  │  Faceted filters, auto-suggest      │
  │  Personalized ranking               │
  └───────────────┬─────────────────────┘
                  │  Clicks product
                  ▼
  ┌─────────────────────────────────────┐
  │         PRODUCT DETAIL PAGE (PDP)   │
  │  Images, description, price         │
  │  Stock availability (real-time)     │
  │  Related / similar products         │
  │  Reviews and ratings                │
  └───────────────┬─────────────────────┘
                  │  Clicks "Add to Cart"
                  ▼
  ┌─────────────────────────────────────┐
  │         CART SERVICE                │
  │  Add item to cart (Redis)           │
  │  Validate: price + stock            │
  │  Apply promotions / coupon codes    │
  │  Show cart summary                  │
  └───────────────┬─────────────────────┘
                  │  Clicks "Proceed to Checkout"
                  ▼
  ┌─────────────────────────────────────┐
  │         CHECKOUT SERVICE            │
  │  Delivery address entry             │
  │  Shipping method selection          │
  │  Tax calculation (by geography)     │
  │  Order summary + final price        │
  └───────────────┬─────────────────────┘
                  │  Clicks "Place Order"
                  ▼
  ┌─────────────────────────────────────┐
  │         PAYMENT SERVICE             │
  │  Tokenize card / wallet             │
  │  Fraud score check                  │
  │  ─────────────────────────          │
  │  FRAUD OK?                          │
  │    YES → Send to payment gateway    │
  │    NO  → Decline / Manual review    │
  │  ─────────────────────────          │
  │  PAYMENT AUTHORIZED?                │
  │    YES → Continue                   │
  │    NO  → Retry / Notify customer    │
  └───────────────┬─────────────────────┘
                  │  Payment Captured
                  ▼
  ┌─────────────────────────────────────┐
  │         ORDER MANAGEMENT (OMS)      │
  │  Create order record                │
  │  Publish: order.placed event        │
  │  Send order confirmation email/SMS  │
  └───────────────┬─────────────────────┘
                  │  Event consumed
                  ▼
  ┌─────────────────────────────────────┐
  │         INVENTORY SERVICE           │
  │  Decrement stock for each item      │
  │  Reserve inventory                  │
  │  Publish: stock.updated event       │
  └───────────────┬─────────────────────┘
                  │
                  ▼
  ┌─────────────────────────────────────┐
  │         FULFILLMENT SERVICE         │
  │  Route order to optimal warehouse   │
  │  (closest, fastest, cheapest)       │
  │  Send pick list to WMS              │
  └───────────────┬─────────────────────┘
                  │
                  ▼
  ┌─────────────────────────────────────┐
  │  WAREHOUSE MANAGEMENT SYSTEM (WMS)  │
  │  Pick items from shelves            │
  │  Pack items                         │
  │  Generate shipping label            │
  └───────────────┬─────────────────────┘
                  │
                  ▼
  ┌─────────────────────────────────────┐
  │         SHIPPING SERVICE            │
  │  Hand off to carrier (FedEx/UPS)    │
  │  Publish: shipment.dispatched event │
  │  Send shipping confirmation + link  │
  └───────────────┬─────────────────────┘
                  │
                  ▼
  ┌─────────────────────────────────────┐
  │         DELIVERY                    │
  │  Carrier delivers to customer       │
  │  Publish: order.delivered event     │
  │  Trigger: loyalty points awarded    │
  │  Trigger: review request email      │
  └─────────────────────────────────────┘

  CUSTOMER receives order ✓

DIAGRAM 4: PAYMENT PROCESSING FLOW
====================================================================

  CUSTOMER submits payment
          │
          ▼
  ┌────────────────────────────────────────┐
  │         CHECKOUT SERVICE               │
  │  Collects: card / wallet / BNPL        │
  │  Tokenizes card (never store raw PAN)  │
  └──────────────┬─────────────────────────┘
                 │
                 ▼
  ┌────────────────────────────────────────┐
  │         FRAUD DETECTION ENGINE         │
  │  Inputs: IP, device, velocity,         │
  │          order amount, location        │
  │  ML model scores transaction           │
  │                                        │
  │  Score > Threshold? ─── YES ──►  BLOCK │
  │       │                        or REVIEW
  │       NO                               │
  └───────┬────────────────────────────────┘
          │  Low risk — proceed
          ▼
  ┌────────────────────────────────────────┐
  │       PAYMENT GATEWAY                  │
  │  (Stripe / Adyen / Braintree)          │
  │                                        │
  │  Send payment token to gateway         │
  │  Gateway contacts card network         │
  │  (Visa / Mastercard / Amex)            │
  └───────┬────────────────────────────────┘
          │
          ▼
  ┌────────────────────────────────────────┐
  │       CARD NETWORK → ISSUING BANK      │
  │                                        │
  │  Check: card valid, funds available,   │
  │         fraud signals                  │
  │                                        │
  │  3DS2 Challenge required?              │
  │    YES → Customer completes OTP / biom │
  │    NO  → Silent authentication         │
  └───────┬────────────────────────────────┘
          │
   ┌──────┴──────────┐
   │                 │
   ▼                 ▼
APPROVED          DECLINED
   │                 │
   ▼                 ▼
┌──────────┐   ┌────────────────────────┐
│ Capture  │   │  Notify customer       │
│ payment  │   │  Suggest retry or      │
│          │   │  alternative payment   │
└────┬─────┘   └────────────────────────┘
     │
     ▼
  ┌────────────────────────────────────────┐
  │       ORDER SERVICE                    │
  │  Mark order as payment_captured        │
  │  Publish: payment.completed event      │
  │  Trigger fulfillment flow              │
  └────────────────────────────────────────┘

                 │
                 ▼

                 │
                 ▼
  ┌─────────────────────────────────┐
  │  ITEM RETURNED TO WAREHOUSE     │
  │  Carrier scans package          │
  │  WMS receives return            │
  └──────────────┬──────────────────┘
                 │
                 ▼
  ┌─────────────────────────────────┐
  │  QUALITY INSPECTION             │
  │                                 │
  │  Item condition check:          │
  │  ┌───────────────────────────┐  │
  │  │ GOOD → Restock inventory  │  │
  │  │ DAMAGED → Write off /     │  │
  │  │           Vendor claim    │  │
  │  │ REFURBISH → Outlet / B2B  │  │
  │  └───────────────────────────┘  │
  └──────────────┬──────────────────┘
                 │
                 ▼
  ┌─────────────────────────────────┐

  │  Trigger refund to original     │
  │  payment method                 │
  │                                 │

  │  • Original payment method      │
  │  • Store credit / gift card     │
  │  • Exchange order creation      │
  └──────────────┬──────────────────┘
                 │
                 ▼
  ┌─────────────────────────────────┐
  │  PAYMENT GATEWAY                │
  │  Process refund                 │
  │  Notify customer via email/SMS  │
  │  Refund appears in 3-5 days     │
  └─────────────────────────────────┘

====================================================================
SUMMARY OF ALL FLOWS
====================================================================

  Flow  1 : High-Level System Overview
            All layers: channels → API GW → services → event bus → data

  Flow  2 : Customer Purchase Journey (Happy Path)
            Browse → Search → PDP → Cart → Checkout → Payment → Order
            → Inventory → Fulfillment → WMS → Ship → Deliver

  Flow  3 : Inventory Management
            Order placed → Reserve stock → Decrement → Reorder trigger
            → PO to vendor → Goods received → Stock replenished

  Flow  4 : Payment Processing
            Checkout → Tokenize → Fraud check → Gateway → Card network
            → Issuing bank → Approve/Decline → Capture → Fulfill

  Flow  5 : Returns & Refunds
            Return request → RMA → Ship back → Inspect → Restock / Write-off
            → Refund to original payment method

  Flow  6 : Personalization & Recommendations
            User behaviour → Clickstream → Feature store → ML model
            → Real-time recommendations on web/app/email

  Flow  7 : Data & Analytics Pipeline
            All sources → Kafka/Batch ingestion → Bronze → Silver → Gold
            → Data warehouse → BI dashboards + ML training

  Flow  8 : Omnichannel BOPIS Flow
            Buy online → Select store → Reserve store stock
            → Alert associate → Ready notification → Customer pickup

  Flow  9 : Supply Chain Flow
            Demand forecast → Reorder trigger → PO → Supplier → Logistics
            → DC receiving → Stock updated → Back-order customers notified

  Flow 10 : Microservices Communication
            Sync (REST/gRPC) for blocking calls
            Async (Kafka events) for decoupled downstream flows

  Flow 11 : Black Friday Scale-Out
            CDN absorbs 80% → Auto-scaling K8s pods → Redis cache
            → Read replicas → Queue-based order processing
